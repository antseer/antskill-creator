# G1 — 数据盘点 · 质量门禁

> **对应 SOP**: `sop/s1_data_inventory.md`
> **执行时机**: S1 结束,进 S2 之前
> **失败处理**: 🔴 项不通过必须回 S1 修正

---

## G1.1 · 穷举完整性(🔴 Critical)

- [ ] demo-v0 每个屏幕每个组件的可见字段都列入盘点
- [ ] 所有静态文案(标题/按钮/Footer/tooltip)已列
- [ ] 所有用户输入(输入框/选择器/开关)已列
- [ ] 所有动态数据(数字/图表/列表/时序)已列
- [ ] 所有 AI 类输出(摘要/建议/推荐理由)已列
- [ ] **状态字段已列**(loading / empty / error,每个至少一条)
- [ ] **交互派生字段已列**(hover/click/filter 后才出现的字段)

## G1.2 · 格式规范(🔴 Critical)

- [ ] 每条记录有稳定 ID(D01+ 格式)
- [ ] ID 只增不改,删除条目保留 ID 标记"(已废弃)"
- [ ] 每条记录包含 9 个字段:ID / 区域 / 元素 / 字段名 / 性质 / 示例值 / 依赖 / 派生逻辑 / 来源
- [ ] 性质字段枚举值合法(用户输入 / 原始值 / 衍生值 / 自然语言 / 静态文案)

## G1.3 · 依赖标注(🔴 Critical)

- [ ] 每个"衍生值"都标注了依赖的 Dxx 编号
- [ ] 每个"衍生值"有至少一句话派生逻辑(不是空白或"—")
- [ ] 依赖链无死循环(Dx 依赖 Dy,Dy 不能反过来依赖 Dx)

## G1.4 · 只列不判铁律(🔴 Critical)

- [ ] **来源列全部填"待定"**,无任何 L1-A / L1-B / L2 / L3 / L4 / L5 标注
- [ ] **无因"MCP 没有"被删除的字段**(S1 不判可行性)
- [ ] 无在派生逻辑字段里写"调 MCP 接口"或"用脚本算"(这是 S2 的事)

## G1.5 · 粒度自查(🟡 Important)

- [ ] 对每个交互(hover/click/filter)手动触发一遍,确认派生字段被捕获
- [ ] 对每个状态切换(default/loading/empty/error)手动切换一遍,确认状态字段被捕获

---

## 常见 Block 点

| 症状 | 对策 |
|---|---|
| 来源列出现 L1-A / L1-B / L2 / L3 / L4 | 全部删除,替换为"待定",门禁重跑 |
| 某字段因 MCP 没有被删除 | **严重违规**。字段必须恢复,放心写入,S2 会归到 L1-B |
| 衍生字段没依赖 | 必须补 Dxx 依赖,否则 S2 的路由无法判定 |
| 派生逻辑写"—"或"TBD" | 至少一句话:"滚动窗口 20 期均值 ± 2σ"这种就行 |
| 漏掉 Loading / Empty / Error | 切换 demo 状态,每个都要至少一条静态文案条目 |

## 通过条件

所有 🔴 Critical 全部通过 → 进入 S2。
