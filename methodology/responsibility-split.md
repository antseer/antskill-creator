### v5 关键边界

1. **AI 不做数学** — L4 接收的是 L1/L2/L3 已经算好的结构化数据(含评分、排名、异常标签),LLM 只负责把这些数据翻译成人话或做主观分类。
2. **L5 不做计算** — 前端只接收上游各层的数据,只负责灌入容器渲染,不在 JS 里做业务计算(纯 UI 映射除外)。
3. **Backend 不写用户文案** — 所有用户可见的自然语言(推荐理由、风险提示、异常说明)由 L4 LLM 产出或 Fallback 模板兜底。
4. **Skill 工厂不碰生产** — 工厂输出半成品(含分层工程目录 + MCP 缺口清单),生产实现全归后端。
5. **L1-B / L2 归后端,不归 Skill 脚本** — 工厂严禁把 L1-B / L2 的缺口"降级塞进 L3 偷偷算"来掩盖。必须登记在 data-prd.md。

## Production-vs-Prototype 二元规则

前端 Demo 和生产实现有本质区别:

| 维度 | Demo(原型) | Production(生产) |
|------|-------------|-------------------|
| 数据 | Mock / 程序生成 | 真实 MCP Tool(L1-A / L1-B / L2) |
| 计算 | 在前端 JS 里 | L3 脚本在后端运行 |
| LLM 调用 | 直接 fetch API | 通过后端代理 + 缓存 + 成本控制 |
| 错误处理 | console.log | 结构化错误码 + 用户友好提示 |
| 认证 | 无 | API Key 加密存储 |
| 并发 | 单用户 | 多用户 + 限流 |

**规则**:
- Demo 的**视觉**和**交互逻辑**是参考标准
- Demo 的**数据获取方式**和**计算位置**不是参考标准
- `skill-prd.md` 附录 D 必须明确标注哪些 Demo 行为是 prototype-only
- `SKILL.md §4 后端续写入口` 必须枚举需要后端改造的 prototype-only 行为

## Retry Contract(错误重试契约)

当任何层级出错时,用户界面必须提供:

### 全局错误卡

```
┌─────────────────────────────────────────┐
│  ⚠ 数据获取失败                          │
│                                         │
│  [数据源名称] 暂时不可用                  │
│  最后成功时间: 2024-01-15 14:30 UTC      │
│                                         │
│  [🔄 重试]   [使用缓存数据]              │
└─────────────────────────────────────────┘
```

### 规则

1. **任何错误都不能让页面白屏** — 至少显示错误卡
2. **重试按钮**必须在首次失败后出现
3. **重试间隔**:1s → 3s → 10s → 停止自动重试,保留手动按钮
4. **缓存降级**:如果有缓存数据,必须提供「使用缓存数据」选项,并标注缓存时间
5. **部分失败**:如果 5 个数据源中 3 个成功 2 个失败,展示成功部分 + 失败部分的错误卡
6. **L4 Fallback**:LLM 调用失败时,必须走 Fallback 模板函数,不弹错误卡(对用户透明)

### 每层降级方案必须在 skill-prd.md 附录 B 中明确列出

## v5 · L1-B / L2 缺口的职责流程

```
Skill 工厂(S2)
  │ 判定某数据为 L1-B(MCP 无此原始数据)或 L2(需新聚合接口)
  ├─→ 写入 data-prd.md 对应条目
  │     - 层级(L1-B / L2)
  │     - 优先级(P0/P1/P2)
  │     - 期望接口形态(endpoint / 入参 / 出参 / 刷新频率)
  │     - 降级策略
  │
  ├─→ 在 skill-prd.md 模块内指向对应数据提需编号
  │
  └─→ 在交付版 SKILL.md frontmatter 的 layers.L1-B / layers.L2 列出

后端工程师(接手)
  │ 按 data-prd.md 的期望接口形态实现
  ├─→ 部署新 MCP Tool
  │
  ├─→ 更新真源仓库并让工厂重新同步缓存
  │
  ├─→ 通知 Skill 开发者移除对应的降级路径(如 L3 脚本里的本地兜底)
  │
  └─→ 通知前端工程师启用相关 UI(如依赖 GAP 的可视化组件)
```

工厂不负责这个流程的后半段,但必须把前半段的交付物做到足够完整,让后端能直接开工。
