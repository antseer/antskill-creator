# S1 — 数据盘点(只列不判)

> **v5 铁律**:S1 只做**穷举**,不做**归属判定**。"MCP 有没有"是 S2 的事。此步混入路由判定 = 门禁不过。

## 阶段目标

产出一份**数据盘点清单**(`data-inventory.md`),穷举 demo-v0 中每一个可见元素的数据需求。

## 入口条件

- S0 产出的需求画布 + demo-v0.html 已通过 G0
- 用户说"开始盘点 / 列数据 / 穷举"

## 方法论

- **对着 demo 逐区域扫**,每个可见元素都要入表。
- **状态字段也要列**(loading / empty / error / hover / click 后出现的字段)。
- **派生字段标明依赖**,只写"依赖 D03",不判断谁来算。
- **来源列统一填"待定"**,禁止写"MCP"/"脚本"/"LLM"。

---

## §1 盘点粒度

必须盘到的元素:
- 所有**静态文案**(标题、按钮文字、Footer、标签文字)
- 所有**用户输入**(输入框、选择器、开关)
- 所有**动态数据**(统计数字、图表数据、列表、时间轴)
- 所有**AI 类输出**(摘要、推荐理由、风险提示,不判断是否 LLM 生成,先列)
- 所有**状态文案**(Loading / Empty / Error)
- 所有**Tooltip / 弹层内容**
- 所有**交互派生字段**(hover 出现的、筛选后才有的、点击后展开的)

## §2 盘点清单结构

按**区域 → 元素 → 数据需求**三层组织。每条记录必须包含:

| 字段 | 说明 |
|------|------|
| **ID** | D01 / D02 / ... 稳定编号,后续 S2/S3/S4/S5 一直引用 |
| **区域** | 左侧输入 / Hero / 中间可视化 / 信任层 / 状态层 |
| **元素** | 该数据在 UI 上的位置和组件类型 |
| **字段名** | 一句话描述这是什么数据 |
| **性质** | `用户输入` / `原始值` / `衍生值` / `自然语言` / `静态文案` |
| **示例值** | 至少一条 mock 示例 |
| **依赖** | 若为衍生值,列出依赖的 D 编号;否则 `—` |
| **派生逻辑(文字描述)** | 衍生值写一句话逻辑(滚动窗口均值、排序规则、分类判断等);原始值写"直接展示" |
| **来源** | **统一填"待定"**(S2 才判) |

## §3 执行步骤总览

```
1. 打开 demo-v0.html,按区域一一扫描(从上到下,从左到右)
2. 对每个可见元素生成一行记录,分配稳定 ID(D01+)
3. 切换默认态/有数据态/空态,补充状态类字段
4. hover / click / filter 各交互动作跑一遍,补充交互派生字段
5. 对每个衍生值,写一句话派生逻辑(不判归属,只描述怎么得到)
6. 全部来源列填"待定"
7. 产出 data-inventory.md
8. 执行 G1 门禁
```

## §6 常见错误

| 错误 | 纠正 |
|------|------|
| 直接标 L1-A / L1-B / L2 / L3 / L4 | 删掉,S1 不做路由判定;来源列必须统一"待定" |
| 因"MCP 没有"就把字段砍掉 | **严重错误**。S1 不判可行性,缺的字段照常列,S2 才归 L1-B |
| 衍生值没写派生逻辑 | 至少一句话(滚动窗口 20 期的均值加减 K 倍标准差) |
| 漏掉 Tooltip / Loading / Empty | 逐状态切 demo 重新扫 |
| 漏掉 hover / filter / click 后才出现的字段 | 每个交互都要手动触发一遍,补齐派生字段 |
| 派生值依赖关系没写 | 每个衍生字段必须有"依赖 Dxx"标记,否则 S2 无法做路由 |
| ID 不稳定(改了一个字段后全部重排) | ID 只增不改,删除的保留 ID 但标(已废弃) |
